ChronoFlow UI 测试清单（关键路径）

目标：覆盖创建/编辑/删除/提醒等核心用户路径，保证在回归时功能可用。

手工回归与自动化测试项：
1) 创建事件（基本）
   - 操作：打开日视图，点击「+」，输入标题、开始/结束时间、类型，保存。
   - 验证：保存提示出现，事件在日视图对应时段以时间块显示，事件计数更新。

2) 编辑事件
   - 操作：在日/周视图点击某事件，选择「编辑」，修改标题或时间，保存。
   - 验证：保存提示出现，日/周视图显示更新后的信息，时间块位置随时间更新。

3) 删除事件
   - 操作：打开事件详情，选择删除并确认。
   - 验证：事件从日/周/月视图消失，相关提醒被取消（检查 NotificationScheduler）。

4) 提醒（本地）
   - 操作：创建带提醒的事件（提前分钟数），在测试环境模拟时间到达或直接调用调度器。
   - 验证：ReminderReceiver 收到广播并调度通知，点击通知可跳转至对应日期/事件。

5) 月视图点击跨月日期
   - 操作：在月视图点击上/下月显示的日期格。
   - 验证：弹出该日事件列表（若有），并且能跳转到日视图查看详情。

6) 表单校验与错误恢复
   - 操作：尝试用空标题或结束早于开始时间保存。
   - 验证：字段级提示展示并阻止保存；修正后可恢复保存。

7) 可访问性与交互
   - 操作：增大系统字体、切换高对比度主题（如果有）。
   - 验证：文本未被裁剪，交互控件有可触达最小尺寸，颜色对比良好。

自动化优先实现（建议顺序）
- 创建事件基本流程（Compose UI Test）
- 编辑事件断言（Compose UI Test）
- 删除 + 确认通知取消（集成测试 / 单元）
- 提醒触发（集成 / 模拟时间）

附注：上述项可作为手工回归脚本，也可直接转为 Espresso / Compose UI 测试用例。
